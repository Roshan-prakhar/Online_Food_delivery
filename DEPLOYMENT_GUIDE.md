# Food Delivery App - Deployment Guide

## Architecture
- **Backend**: Spring Boot API on Render
- **Frontend**: React apps on Netlify (Customer App + Admin Panel)
- **Database**: PostgreSQL on Render
- **Storage**: Cloudflare R2 for images
- **Payment**: Razorpay

## Step 1: Deploy Backend (Spring Boot API)

### 1.1 Push to GitHub
```bash
git add .
git commit -m "Ready for deployment"
git push origin main
```

### 1.2 Deploy on Render
1. Go to [render.com](https://render.com)
2. Connect your GitHub repository
3. Select `foodiesapi` folder
4. Render will automatically detect the `render.yaml` file
5. Deploy with the following configuration:
   - **Web Service**: Spring Boot
   - **Database**: PostgreSQL (auto-created)
   - **Environment Variables**: Auto-configured

### 1.3 Set Environment Variables on Render
After deployment, set these in Render dashboard:
```
JWT_SECRET=your-jwt-secret-key-here
CLOUDFLARE_ACCESS_KEY=your-r2-access-key
CLOUDFLARE_SECRET_KEY=your-r2-secret-key
CLOUDFLARE_R2_ENDPOINT=https://your-account-id.r2.cloudflarestorage.com
CLOUDFLARE_BUCKET_NAME=your-bucket-name
RAZORPAY_KEY=your-razorpay-key
RAZORPAY_SECRET=your-razorpay-secret
```

## Step 2: Deploy Frontend Apps

### 2.1 Customer App (Foodies)
1. Go to [netlify.com](https://netlify.com)
2. Connect your GitHub repository
3. Select `foodies` folder
4. Build settings:
   - **Build command**: `npm run build`
   - **Publish directory**: `dist`
   - **Node version**: 18

### 2.2 Admin Panel
1. Add new site on Netlify
2. Select `adminpanel` folder
3. Build settings:
   - **Build command**: `npm run build`
   - **Publish directory**: `dist`
   - **Node version**: 18
4. **Environment Variables** (in Netlify dashboard):
   - `VITE_API_URL`: `https://your-render-app-url.onrender.com/api`

## Step 3: Update Frontend API URLs

### 3.1 Update Foodies App
Update API base URL in `foodies/src/service/api.js`:
```javascript
const API_BASE_URL = 'https://your-render-app-url.onrender.com/api';
```

### 3.2 Update Admin Panel
The admin panel now uses centralized API configuration. Set the environment variable in Netlify:
- **Variable Name**: `VITE_API_URL`
- **Value**: `https://your-render-app-url.onrender.com/api`

The configuration is in `adminpanel/src/config/api.js` and will automatically use the production URL when deployed.

## Step 4: Configure Cloudflare R2

### 4.1 Create R2 Bucket
1. Go to Cloudflare Dashboard
2. Navigate to R2 Object Storage
3. Create bucket: `foodies-images`
4. Generate API tokens

### 4.2 Set Bucket Public Access
1. Go to bucket settings
2. Enable public access
3. Note the public URL: `https://pub-bucket-name.r2.dev`

## Step 5: Configure Razorpay

### 5.1 Create Razorpay Account
1. Go to [razorpay.com](https://razorpay.com)
2. Create account
3. Get API keys from Dashboard → Settings → API Keys

## Step 6: Test the Application

### 6.1 Test API Endpoints
```bash
# Health check
curl https://your-app.onrender.com/actuator/health

# Test user registration
curl -X POST https://your-app.onrender.com/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{"name":"Test User","email":"test@example.com","password":"password123"}'
```

### 6.2 Test Frontend
1. Visit your Netlify URLs
2. Test user registration/login
3. Test food ordering flow
4. Test admin panel functionality

## Environment Variables Summary

### Render (Backend)
- `DATABASE_URL`: Auto-generated by Render
- `DB_USERNAME`: Auto-generated by Render
- `DB_PASSWORD`: Auto-generated by Render
- `JWT_SECRET`: Generate secure random string
- `CLOUDFLARE_ACCESS_KEY`: From R2 dashboard
- `CLOUDFLARE_SECRET_KEY`: From R2 dashboard
- `CLOUDFLARE_R2_ENDPOINT`: Your R2 endpoint
- `CLOUDFLARE_BUCKET_NAME`: Your bucket name
- `RAZORPAY_KEY`: From Razorpay dashboard
- `RAZORPAY_SECRET`: From Razorpay dashboard

### Netlify (Frontend)
- No environment variables needed for basic deployment

## URLs Structure
- **API**: `https://foodies-api.onrender.com/api`
- **Customer App**: `https://foodies-customer.netlify.app`
- **Admin Panel**: `https://foodies-admin.netlify.app`

## Admin Panel Features
- **Food Management**: Add, edit, delete food items
- **Image Upload**: Upload food images via Cloudflare R2
- **Order Management**: View all orders, update order status
- **Dashboard**: Real-time order tracking and management

## Troubleshooting

### Common Issues
1. **Build fails**: Check `render.yaml` syntax
2. **Database connection**: Verify DATABASE_URL format
3. **CORS issues**: Ensure CORS is configured in Spring Boot
4. **Image upload fails**: Check R2 credentials and bucket permissions

### Health Check
Always test: `https://your-app.onrender.com/actuator/health`
